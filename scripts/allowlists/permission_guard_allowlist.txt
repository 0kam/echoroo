# Phase 2.11 P1-a: line-level fingerprint format. Each non-comment
# line is <file>:<function_name>:missing-permission-guard. The
# fingerprint is locked to a SPECIFIC violation so adding a new
# unguarded endpoint to a file that already has allowlisted entries
# fails the lint (the previous file-level allowlist silently inherited
# every new violation in the same file).
#
# Phase 3 must drive each entry to empty before final cleanup.
# Adding a new entry requires a justifying PR comment.
#
# Inline comments use TWO SPACES + #. Lines beginning with #
# are full-line comments. See scripts/allowlists/README.md.
