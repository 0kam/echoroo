# 機能仕様書: Explore・レポート・エクスポート

**フィーチャーブランチ**: `005-explore-reports`
**作成日**: 2026-03-02
**ステータス**: ドラフト
**入力**: Exploreビュー、データセット詳細強化、サンプリングレビュー、エクスポート、共有

## ユーザーシナリオとテスト *(必須)*

### ユーザーストーリー 1 - Exploreビュー (優先度: P2)

ユーザーはExploreページを開くと、H3マップにプロジェクト横断でサイトが表示される。種名で検索すると、該当する検出があるサイトがハイライトされ、クリックでデータセットへ遷移できる。

**優先度の理由**: プロジェクト横断でのデータ探索はプラットフォームの差別化要素。共有データの発見性を高める。

**独立テスト**: Exploreページを開き、サイト表示・種名検索・フィルタ切り替え・データセット遷移を確認。

**受け入れシナリオ**:

1. **Given** 複数プロジェクトにサイトとデータセットが存在する状態で、**When** Exploreページを開く、**Then** H3マップに全サイトが六角形セルで表示される
2. **Given** Exploreページで、**When** 種名（学名または一般名）で検索する、**Then** 該当種の検出があるサイトのみがハイライトされる
3. **Given** 検索結果が表示された状態で、**When** 「確認済みのみ」フィルタを切り替える、**Then** confirmed状態のアノテーションがあるサイトのみに絞り込まれる
4. **Given** マップ上のサイトをクリックした状態で、**When** サイト情報ポップアップが表示される、**Then** サイト名・プロジェクト名・検出数が表示され、データセットへの遷移リンクがある
5. **Given** ユーザーが閲覧権限のないプロジェクトのサイトに対して、**When** アクセスを試みる、**Then** 公開プロジェクトのサイトのみが表示される

---

### ユーザーストーリー 2 - Spiral Plot (優先度: P2)

データセット詳細画面にSpiral Plot（時刻×日付のヒートマップ）が表示され、種の検出パターン（朝のコーラス、夜行性など）が一目で分かる。

**優先度の理由**: 検出パターンの可視化は調査報告書やデータ理解に直結する。テキストデータだけでは読み取れないパターンを発見できる。

**独立テスト**: 検出結果があるデータセットの詳細画面を開き、Spiral Plotが正しく描画されることを確認。

**受け入れシナリオ**:

1. **Given** 検出結果があるデータセットの詳細画面で、**When** Spiral Plotセクションを表示する、**Then** Y軸=時刻（0-24h）、X軸=日付のヒートマップが描画される
2. **Given** Spiral Plotが表示されている状態で、**When** 種フィルタを選択する、**Then** 選択した種の検出パターンのみが表示される
3. **Given** Spiral Plotが表示されている状態で、**When** セルをホバーする、**Then** 日付・時刻・検出数がツールチップで表示される
4. **Given** 検出結果がないデータセットで、**When** Spiral Plotセクションを表示する、**Then** 「検出結果がありません」メッセージが表示される
5. **Given** データセット詳細画面で、**When** レビュー進捗セクションを確認する、**Then** 確認済み/未レビュー/未処理の割合が表示される
6. **Given** データセット詳細画面で、**When** Species Listセクションを確認する、**Then** 種ごとの検出数と確認状況が一覧表示される

---

### ユーザーストーリー 3 - サンプリングレビュー (優先度: P2)

ユーザーがデータセットで「サンプリングレビュー」を実行すると、確認済み区間を除外したランダムな時間セグメントが提示され、効率的にネガティブデータを作成できる。

**優先度の理由**: ML学習に不可欠なネガティブデータの作成を効率化する。手動で全録音を聴くのは非現実的なため、ランダムサンプリングが必要。

**独立テスト**: サンプリングレビューを開始し、セグメントの聴取・アノテーション追加・確認済みマーク操作を確認。

**受け入れシナリオ**:

1. **Given** データセット詳細画面で、**When** 「サンプリングレビュー」ボタンをクリックする、**Then** サンプリング設定画面が表示される（セグメント数、セグメント長を設定可能）
2. **Given** サンプリング設定後、**When** レビューを開始する、**Then** 確認済み区間を除外したランダムな時間セグメントが生成される
3. **Given** セグメントが表示されている状態で、**When** スペクトログラムと音声を確認する、**Then** セグメント内の音声が再生・表示される
4. **Given** セグメント内に種の鳴き声を発見した状態で、**When** アノテーションを追加する、**Then** start_time/end_time/種タグ付きのアノテーションが作成される
5. **Given** セグメントの確認が完了した状態で、**When** 「確認済み」ボタンをクリックする、**Then** ConfirmedRegionが作成され、次のセグメントに進む
6. **Given** 全セグメントの確認が完了した状態で、**When** セッションが終了する、**Then** レビュー結果サマリー（確認数・アノテーション追加数）が表示される

---

### ユーザーストーリー 4 - データエクスポート (優先度: P2)

ユーザーは検出結果CSVと、ML学習用データセット（音声クリップ + annotations.csv + metadata.json）をエクスポートできる。

**優先度の理由**: 調査報告書の作成とML学習データの再利用は、ユーザーの最終的な成果物に直結する。

**独立テスト**: エクスポートを実行し、出力ファイルのフォーマットとデータの整合性を確認。

**受け入れシナリオ**:

1. **Given** 検出結果があるデータセットで、**When** 「検出結果CSVエクスポート」を選択する、**Then** VISION.md準拠のCSVがダウンロードされる
2. **Given** エクスポートCSVを確認した時、**When** カラムを検証する、**Then** recording_filename, start_time, end_time, species, confidence, source, model_name, model_version, verified, verified_by, search_query_* が含まれる
3. **Given** 確認済みアノテーションが存在するデータセットで、**When** 「ML学習用エクスポート」を選択する、**Then** audio/ + annotations.csv + metadata.json + README.txt のZIPがダウンロードされる
4. **Given** ML学習用エクスポートで、**When** ポジティブデータを確認する、**Then** 人が範囲指定したアノテーション区間の音声クリップが含まれる
5. **Given** ML学習用エクスポートで、**When** ネガティブデータを確認する、**Then** ConfirmedRegionでアノテーションがない区間の音声クリップが含まれる
6. **Given** エクスポート処理中に、**When** 進捗を確認する、**Then** 処理状況が表示される（大規模データの場合はバックグラウンド処理）

---

### ユーザーストーリー 5 - 検出結果の共有 (優先度: P3)

プロジェクト間で検出結果（生の音声データは含まない）を共有できる。受信者は確認済みデータのみを閲覧可能。

**優先度の理由**: プラットフォーム内でのデータ共有は将来的な差別化要素だが、コアワークフロー完成後に実装可能。

**独立テスト**: 共有リンクを作成し、別プロジェクトのユーザーが検出結果を閲覧できることを確認。

**受け入れシナリオ**:

1. **Given** プロジェクト管理者として、**When** データセットの検出結果共有を設定する、**Then** 共有対象プロジェクト/ユーザーを選択できる
2. **Given** 共有が設定された状態で、**When** 受信者がExploreビューで検索する、**Then** 共有されたデータセットの確認済み検出結果が表示される
3. **Given** 共有された検出結果を閲覧中に、**When** 詳細を確認する、**Then** 種名・日時・信頼度・確認状況が表示されるが、音声データにはアクセスできない
4. **Given** 共有された検出結果で、**When** 未確認の検出結果を表示しようとする、**Then** 確認済みの結果のみが表示される
5. **Given** 未認証ユーザーが、**When** 共有データにアクセスしようとする、**Then** ログインが要求される

---

### エッジケース

- 大量のサイト（1000+）がある場合のマップパフォーマンスは？ → H3のクラスタリングで表示を最適化。ズームレベルに応じて解像度を調整
- サンプリングで全区間が確認済みの場合どうなるか？ → 「全区間がレビュー済みです」メッセージを表示
- エクスポート対象が大量（10000+アノテーション）の場合どうなるか？ → バックグラウンドジョブとして処理し、完了通知を送信
- Spiral Plotで日付範囲が1年以上の場合どうなるか？ → 横軸をスクロール可能にするか、月単位に集約
- 共有先プロジェクトが削除された場合どうなるか？ → 共有設定も自動的に削除
- サンプリングレビュー中にセッションが切れた場合どうなるか？ → 確認済みセグメントは保存済み。未確認セグメントは次回セッションで再生成

## 要件 *(必須)*

### 機能要件

#### Exploreビュー
- **FR-001**: システムは、全プロジェクトのサイトをH3マップ上に表示できなければならない
- **FR-002**: システムは、種名（学名・一般名）によるサイト検索をサポートしなければならない
- **FR-003**: システムは、「確認済みのみ」と「全検出結果」のフィルタ切り替えをサポートしなければならない
- **FR-004**: システムは、サイトクリック時にデータセットへの遷移リンクを表示しなければならない
- **FR-005**: システムは、ユーザーの権限に基づいてサイトの可視性を制御しなければならない

#### Spiral Plot・データセット詳細強化
- **FR-006**: システムは、時刻×日付のヒートマップ（Spiral Plot）をCanvas APIで描画できなければならない
- **FR-007**: システムは、Spiral Plotに種フィルタを適用できなければならない
- **FR-008**: システムは、Spiral Plotのセルホバーでツールチップを表示できなければならない
- **FR-009**: システムは、データセットのレビュー進捗（confirmed / unreviewed / unprocessed）を表示できなければならない
- **FR-010**: システムは、データセットの種別検出数と確認状況を一覧表示できなければならない

#### サンプリングレビュー
- **FR-011**: システムは、確認済み区間を除外したランダムな時間セグメントを生成できなければならない
- **FR-012**: システムは、セグメント数とセグメント長の設定をサポートしなければならない
- **FR-013**: システムは、セグメント内の音声再生とスペクトログラム表示をサポートしなければならない
- **FR-014**: システムは、セグメント確認時にConfirmedRegionを作成できなければならない
- **FR-015**: システムは、セグメント内でのアノテーション追加をサポートしなければならない
- **FR-016**: システムは、サンプリングレビューのセッション結果サマリーを表示できなければならない

#### データエクスポート
- **FR-017**: システムは、VISION.md準拠の検出結果CSVをエクスポートできなければならない
- **FR-018**: システムは、ML学習用データセット（audio/ + annotations.csv + metadata.json + README.txt）をZIP形式でエクスポートできなければならない
- **FR-019**: システムは、ポジティブデータ（確認済みアノテーション区間の音声クリップ）を生成できなければならない
- **FR-020**: システムは、ネガティブデータ（ConfirmedRegionでアノテーションなしの区間）を生成できなければならない
- **FR-021**: システムは、大規模エクスポートをバックグラウンドジョブとして処理できなければならない

#### 共有
- **FR-022**: システムは、プロジェクト間で検出結果（音声データ除外）を共有できなければならない
- **FR-023**: システムは、共有受信者に確認済み検出結果のみを表示しなければならない
- **FR-024**: システムは、共有を認証済みユーザー間に限定しなければならない

### 主要エンティティ

- **ConfirmedRegion（確認済み区間）**: 人がレビューした録音上の時間区間。recording_id, start_time, end_time, reviewed_by_id, created_at を持つ。アノテーションあり=ポジティブ、なし=ネガティブ
- **SamplingSession（サンプリングセッション）**: サンプリングレビューの1回のセッション。dataset_id, user_id, segment_count, segment_duration, status を持つ
- **SamplingSegment（サンプリングセグメント）**: セッション内の個別セグメント。recording_id, start_time, end_time, status（pending/reviewed）を持つ
- **DetectionShare（検出結果共有）**: プロジェクト間の共有設定。source_dataset_id, target_project_id, shared_by_id, confirmed_only を持つ
- **ExportJob（エクスポートジョブ）**: バックグラウンドエクスポート処理。dataset_id, format, status, file_path を持つ

## 成功基準 *(必須)*

### 測定可能な成果

- **SC-001**: Exploreマップが1000サイトを3秒以内にレンダリングできる
- **SC-002**: 種名検索が500ms以内に結果を返す
- **SC-003**: Spiral Plotが10,000検出ポイントを2秒以内に描画できる
- **SC-004**: サンプリングセグメント生成が100セグメントに対して1秒以内に完了する
- **SC-005**: 検出結果CSVエクスポートが10,000レコードに対して5秒以内に完了する
- **SC-006**: ML学習用エクスポートが1,000クリップに対して5分以内に完了する
- **SC-007**: 共有された検出結果の表示が2秒以内に完了する

## 前提条件

- 003-annotation（新アノテーションモデル）と004-ml-pipeline（ML検出実行）が実装済み
- ConfirmedRegionモデルが003または本機能で実装済み
- DetectionRunモデルとAnnotation（新モデル）が利用可能
- H3マップコンポーネント（H3MapPicker.svelte）が利用可能
- 音声ファイルへのサーバーサイドアクセスが可能（エクスポート用）
