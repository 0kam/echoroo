# Echoroo プロダクトビジョン

> English version: [VISION.md](VISION.md)

## ミッション

NPO、自治体、研究者が環境音データを管理し、
MLの力を借りて野生生物調査を行えるようにする。
生物音響学や機械学習の専門知識は不要。

## ターゲットユーザー

**メインターゲット**: 野生生物調査を行うNPO・自治体の担当者

- 音響・MLの専門家ではない
- AudioMoth等の自動録音機をフィールドに設置してデータを収集している
- 「この場所に、いつ、どんな種がいるか」を知りたい
- ステークホルダー向けの報告書を作成する必要がある
- 時間も予算も限られている

## コア原則

1. **ML主導**: ユーザーはゼロからアノテーションするのではなく、MLの結果をレビューする
2. **設定不要**: データを入れたら結果が出てくる
3. **トレーサビリティ**: すべての検出結果の出処を追跡可能（モデル名、バージョン、信頼度、人による確認状況）
4. **ネガティブデータの重要性**: 「何も見つからなかった」と「まだ確認していない」を区別する

## データモデル

### 階層構造

```
プロジェクト（調査プロジェクト。例: 「2026年 谷戸山公園モニタリング」）
  └── サイト（設置場所。H3六角形セルで地理的に表現）
       └── データセット（1台のレコーダーの1回のデプロイメント分 = 音声ファイルのディレクトリ）
            └── 録音（メタデータ付きの単一音声ファイル）
```

### アノテーション（録音に直接紐づく）

アノテーションは**録音上の時間セグメント**であり、固定長クリップには依存しない。

```
録音
  ├── アノテーション（start_time, end_time, 種タグ, ソース, 信頼度, ステータス）
  └── 確認済み区間（start_time, end_time — 人がレビューした時間区間を記録）
```

- **アノテーション**: 「時刻Y〜Zに種Xが検出された」
  - ソース: birdnet / perch_search / human
  - ステータス: unreviewed（未確認）/ confirmed（確認済み）/ rejected（棄却）
  - 信頼度: 0.0〜1.0（ML検出の場合）
  - オプション: freq_low, freq_high（専門家向け、周波数帯域）

- **確認済み区間（ConfirmedRegion）**: 「人がこの時間区間をレビューした」
  - アノテーションあり = ポジティブデータ
  - アノテーションなし = ネガティブデータ（明示的に「何もなかった」）
  - 未確認区間 = 不明

### 検出実行記録（DetectionRun）— 内部データ、ユーザーには非公開

MLの実行メタデータをトレーサビリティのために保持:
- モデル名、バージョン、パラメータ
- 処理ステータス、件数
- アノテーションとソース実行の紐づけ

### クリップ — 内部処理のみ

クリップは**ユーザーには公開しない**。ML推論時にメモリ上で動的に生成する
（BirdNETは3秒、Perchは5秒など、モデルごとに異なる入力長に対応）。

### 廃止する概念

以下のWhombat由来の概念は**廃止**する:
- アノテーションプロジェクト（AnnotationProject）
- アノテーションタスク（AnnotationTask）
- クリップアノテーション（ClipAnnotation）
- サウンドイベントアノテーション（SoundEventAnnotation → Annotationに置き換え）
- ユーザー向けクリップ

## コアワークフロー

### 1. データインポート + 自動処理（バックグラウンド）

```
ユーザーがデータセットをインポート（音声ファイルのディレクトリ）
  ↓（自動実行、ノンブロッキング、Celeryワーカー）
  ├── BirdNET検出 → アノテーションとして保存（source=birdnet）
  └── Perch埋め込み生成 → pgvectorに保存
  ↓
ユーザーに通知「処理完了しました」
```

- MLモデルの選択は管理者が設定。ユーザーが実行ごとに選ぶことはしない。デフォルトは以下の通り。
- 検出: BirdNET（鳥類の種同定）
- 埋め込み: Perch（BirdNET対象外の種の類似音検索用）
- 処理は完全にノンブロッキング。ユーザーは処理中も他の作業が可能

### 2. ML検出結果のレビュー（ユーザーのメイン作業）

```
Species List View（デフォルト画面）
  → 検出された全種を、件数・信頼度とともに一覧表示
  → 種をクリック → カード型レビュービュー
  → 各検出に対して:
     1. スペクトログラム上でML検出区間がハイライト表示される
     2. ユーザーが聴いて/見て確認
     3. 正しい場合: 鳴き声の実際の時間範囲をドラッグで指定 + 種を確認
        誤検出の場合: Reject（ノイズ）
        種が違う場合: 正しい種を選択 + 時間範囲を指定
```

レビューフロー:
- ML検出結果（クリップ単位）は「下敷き」として表示
- ユーザーは**必ず鳴き声の実際の時間範囲を指定**してから確認する
- これにより、クリップ長に依存しない精密なアノテーションが作成される
- 結果: ML検出とは独立した、人が確認した時間セグメントのアノテーション

UI設計原則:
- Species Listがエントリーポイント（「何が見つかったか？」に直接回答）
- ミニスペクトログラム+再生ボタン付きのカードグリッド
- ML検出区間をハイライト表示し、その中で範囲選択を促す
- デフォルトでは周波数軸の操作なし
- パワーユーザー向けにキーボードショートカット対応

### 3. 類似音検索（BirdNET対象外の種を探す）

```
ユーザーが参照音を選択 or アップロード
  → Perch埋め込みでデータセット全体を類似音検索
  → 候補が検出結果として提示される
  → ユーザーがレビュー: 確認 / 棄却
```

ユースケース:
- 哺乳類、両生類、昆虫（BirdNETがカバーしていない種）
- BirdNETの学習データにない希少種・地域固有種
- 「この音は何？」という探索的な利用

### 4. サンプリングによる手動レビュー

```
ユーザーがデータセットで「サンプリングレビュー」をクリック
  → システムがランダムな時間セグメントを生成（確認済み区間は除外）
  → ユーザーが各セグメントを聴く
  → 種の鳴き声があれば: アノテーションを追加
  → セグメントを確認済みとしてマーク（ConfirmedRegionを作成）
```

目的:
- ML学習用のネガティブデータの作成
- ML結果の品質チェック
- MLが見逃した種の発見

## 結果表示

### Explore（プロジェクト横断検索）

- H3六角形セルでサイトを地図上に表示
- 種名で検索
- フィルター: 「確認済みのみ」/「全検出結果」
- 結果: マッチしたサイト一覧 → クリックでデータセットへ遷移

### データセット詳細画面

- **Spiral Plot**: 検出結果の時間パターン（時刻×日付のヒートマップ）
- **レビュー進捗**: 確認済み / 未レビュー / 未処理
- **Species List**: 種ごとの検出数・確認状況
- 個別の録音への導線

## エクスポート

### 1. 検出結果CSV（調査レポート向け）

```csv
recording_filename, start_time, end_time, species, confidence,
source, model_name, model_version, verified, verified_by,
search_query_recording, search_query_start_time, search_query_end_time
```

フィールド:
- **source**: `birdnet` / `perch_search` / `human`
- **verified**: `true` / `false` / `null`（未確認）
- **search_query_***: 埋め込み検索の場合、クエリ元の音源情報

### 2. ML学習用データセット（モデル開発向け）

確認済みデータを、機械学習にそのまま利用可能な形でエクスポートする。

```
export/
  ├── audio/                      # 音声ファイル（区間ごとに切り出し）
  │   ├── rec001_12.3-14.8.wav    # ポジティブ: 鳴き声あり
  │   ├── rec002_0.0-15.0.wav     # ネガティブ: 確認済み・鳴き声なし
  │   └── ...
  ├── annotations.csv             # 各音声のラベル
  │   recording, start, end, species, is_positive
  │   rec001, 12.3, 14.8, Parus_minor, true
  │   rec002, 0.0, 15.0, , false
  ├── metadata.json               # エクスポート条件、モデル情報、
  │                               # サンプリング条件等
  └── README.txt                  # データセットの説明
```

内容:
- **ポジティブデータ**: 人が範囲指定して確認したアノテーション付き音声クリップ
- **ネガティブデータ**: ConfirmedRegionでアノテーションがない区間の音声クリップ
- **メタデータ**: 元のML検出結果、サンプリング条件、モデル情報を含む
- サンプリングレビュー由来のデータも含む（サンプリングされた音源 + そのアノテーション）

### 3. 追加形式（将来）

- **PDFレポート**: ステークホルダー向けの整形された調査報告書
  - 種チェックリスト、活動タイムライン、サイト地図
  - 行政・NPOの報告書としてそのまま利用可能
- **JSON**: 機械可読な構造化データ

## 共有

- **検出結果のみの共有**: 生の音声データは公開せず、「いつ・どこで・何の種が」をプロジェクト間で共有
- **確認済みフィルター**: 受信者は人が確認した検出結果のみ閲覧可能
- **未認証アクセスなし**: 共有はプラットフォーム内（認証済みユーザー間）のみ
- **データダウンロード**: 権限のあるユーザー向けのエクスポート強化


## 管理者設定

- **MLモデル**: 実行するモデルの選択（検出=BirdNET、埋め込み=Perch）
- **ユーザー管理**: ロール、有効化/無効化、メール認証
- **システム設定**: 登録モード、セッションタイムアウト
- **レコーダーカタログ**: デバイスタイプ（AudioMoth等）
- **ライセンスカタログ**: データセット向けCCライセンス
